Hermes Agent 技能学习笔记 - 第1天：Creative 分类详解

计划：6月1日-6月5日学完全部92个技能，每天发布学习报告。本文是第1天，涵盖 Creative 分类下全部20个技能。

1. architecture-diagram（架构图生成）

用途：生成深色主题的 SVG 架构/云/基础设施图，输出为独立的 HTML 文件。

核心特点：

  • 无需外部工具、API密钥或渲染库
  • 支持前端、后端、数据库、云服务、安全组件等多种图标类型
  • 每个组件类型有固定的颜色编码（如前端=青色，后端=绿色，数据库=紫色）
  • 使用 JetBrains Mono 等宽字体，40px网格背景

输出：一个自包含的 .html 文件，双击即可在浏览器中打开，离线可用。

2. ascii-art（ASCII艺术生成）

用途：在终端和文档中生成字符艺术。

工具集：

工具 功能 示例
pyfiglet 文字大banner，571种字体 python3 -m pyfiglet "HELLO" -f slant
cowsay 气泡消息框+ASCII角色 cowsay -f tux "Linux"
boxes 装饰性边框框架 echo "Hi" | boxes -d stone
toilet 彩色文字艺术 toilet --gay "Rainbow"
ascii-image-converter 图片转ASCII字符画 ascii-image-converter photo.png -C

特殊技巧：可以用 curl -s "qrenco.de/URL" 生成二维码的ASCII形式，或 curl -s "wttr.in/London" 显示天气ASCII图。

3. ascii-video（ASCII视频制作）

用途：将视频/音频转换为彩色ASCII字符视频（MP4/GIF格式）。

核心能力：

  • 视频转ASCII：还原视频内容的字符画版本
  • 音频驱动：基于音频特征（FFT、节拍）生成可视化动画
  • 程序生成：通过数学效果（波纹、粒子、分形）创作ASCII动画
  • 混音模式：视频+音频驱动叠加

技术栈：Python 3.10+、NumPy、SciPy、Pillow、ffmpeg

关键概念：使用自适应色调映射（percentile-based adaptive brightness normalization），而非简单的乘法来保持ASCII字符的可见性。

4. baoyu-article-illustrator（文章配图生成）

用途：为文章自动生成配图，保持类型×风格×调色板的一致性。

三个维度：

维度 选项 说明
Type（类型） infographic, scene, flowchart, comparison, framework, timeline 信息结构
Style（风格） notion, warm, minimal, blueprint, watercolor, elegant 渲染方式
Palette（调色板） macaron, warm, neon 颜色方案（可选）

工作流程：分析文章内容 → 确认配图方案 → 生成大纲 → 生成提示词 → 生成图片 → 插入文章

5. baoyu-comic（知识漫画创作）

用途：创建知识漫画（教育漫画、传记漫画、教程漫画）。

艺术风格选项：ligne-claire（默认）、manga、realistic、ink-brush、chalk、minimalist

色调选项：neutral（默认）、warm、dramatic、romantic、energetic、vintage、action

预设主题：

  • ohmsha：日式漫画风格，视觉隐喻丰富
  • wuxia：武侠风格，墨刷+动作
  • shoujo：少女漫画风格，装饰性元素多
  • four-panel：四格漫画，起承转合结构

注意：image_generate 是纯提示词工具（输入prompt+宽高比，返回图片URL），不支持参考图片输入。需要用户通过文本描述来提取风格特征。

6. baoyu-infographic（信息图生成）

用途：21种布局 × 21种风格的高密度信息可视化图。

21种布局：

  • 线性时间线（linear-progression）、二元对比（binary-comparison）、层级金字塔（hierarchical-layers）、中心辐射（hub-spoke）、漏斗转化（funnel）、仪表盘（dashboard）、周期表（periodic-table）…

21种风格：

  • 手工艺感（craft-handmade）、黏土动画（claymation）、赛博朋克霓虹（cyberpunk-neon）、企业孟菲斯（corporate-memphis）、技术蓝图（technical-schematic）、乐高积木（lego-brick）、宜家说明书（ikea-manual）…

推荐组合：

  • 教程类：linear-progression + ikea-manual
  • 技术文档：structural-breakdown + technical-schematic
  • 高密度信息：dense-modules + pop-laboratory

7. claude-design（HTML设计原型）

用途：设计一次性的HTML产物（着陆页、原型、演示文稿、组件库）。

与 popular-web-designs 的区别：

  • claude-design：设计流程和审美判断——如何根据简报生成变体、验证本地HTML产物、避免AI设计垃圾
  • popular-web-designs：54个现成的设计系统（Stripe、Linear、Vercel等）的精确CSS/颜色值

核心原则：

  1. 从上下文出发，不凭直觉——先阅读现有的设计文件、品牌文档、截图
  2. 设计系统先行——颜色、字体、间距、圆角、阴影、动效
  3. 至少三个变体：保守型、最佳匹配型、突破型
  4. 避免填充内容（虚假指标、装饰性SVG、无意义的图表）

输出：自包含HTML文件，可直接双击在浏览器打开。

8. comfyui（AI图像/视频/音频生成）

用途：通过 ComfyUI 生成图片、视频、音频和3D内容。

两层架构：

Layer 1: comfy-cli（官方生命周期工具）
  → 安装、启动、停止、自定义节点管理、模型下载

Layer 2: REST/WebSocket API + skill scripts
  → 工作流执行、参数注入、进度监控、输出下载

本地 vs 云端选择：

  • Comfy Cloud：托管服务，RTX 6000 Pro GPU，零安装，但免费账户无法通过API运行
  • 本地安装：免费，需要NVIDIA GPU ≥6GB VRAM（SDXL需≥8GB，Flux/video需≥12GB）

核心脚本：

  • hardware_check.py：探测GPU/VRAM/磁盘，建议本地或云端
  • run_workflow.py：注入参数、提交任务、监控、下载结果
  • check_deps.py：检查工作流的节点/模型依赖是否满足

注意：工作流必须使用API格式（每个节点有class_type），而非编辑器格式（顶级nodes和links数组）。

9. creative-ideation（创意构思）

用途：通过创意约束生成项目灵感。

核心规则：约束 + 方向 = 创造力。没有约束就没有创造力。

约束库示例：

开发者专用：

  • “解决自己的痛点”：构建你本周希望存在的工具，50行以内，今天发布
  • “自动化烦人的事”：找到工作流中最繁琐的部分，写脚本解决
  • “应该存在的CLI工具”：git undo-that-thing、docker why-is-this-broken

艺术家专用：

  • “复制某个东西”：挑选你崇拜的作品，从头重建，学习藏在差距里的东西
  • “一百万个某物”：一百万是很多也很少——探索规模效应

任何人：

  • “敌意UI”：故意让人痛苦地使用的东西
  • “从爆点开始”：想一个有趣的句子，倒推让它成真

10. design-md（设计令牌规范）

用途：编写、验证、导出 Google DESIGN.md 格式的设计令牌规范文件。

文件结构：

---
version: alpha
name: 品牌名称
description: 设计理念描述
colors:
  primary: "#1A1C1E"
  secondary: "#6C7278"
typography:
  h1:
    fontFamily: Public Sans
    fontSize: 3rem
    fontWeight: 700
components:
  button-primary:
    backgroundColor: "{colors.tertiary}"
    textColor: "#FFFFFF"
---
## Overview（正文说明）

工具：npx @google/design.md lint DESIGN.md 验证结构和WCAG对比度，export --format tailwind 导出为Tailwind主题。

核心原则：

  • 不要嵌套组件变体：button-primary.hover 是错的，button-primary-hover 是对的
  • 十六进制颜色必须加引号："#1A1C1E" 而非 #1A1C1E
  • 令牌引用用点分隔路径：{colors.primary} 而非 {primary}

11. excalidraw（手绘风格图表）

用途：创建手绘风格的 Excalidraw JSON 图表文件，可拖放到 excalidraw.com 打开编辑。

核心特点：

  • 无账号、无API密钥、无渲染库，只需要JSON
  • 生成 .excalidraw 文件，双击即可在excalidraw.com查看和编辑

元素类型：矩形、椭圆、菱形、文本、箭头、绑定箭头

关键技巧：

  • 标签必须使用容器绑定方法：形状有boundElements列出文本，文本有containerId指回形状
  • 绝对不要在形状上用 "label": {...} ——这不是有效属性会被静默忽略
  • 字体最小16px（正文标签）、20px（标题）、14px（次要注释）

颜色编码：

用途 填充色 十六进制
主要/输入 浅蓝 #a5d8ff
成功/输出 浅绿 #b2f2bb
警告/外部 浅橙 #ffd8a8
处理/特殊 浅紫 #d0bfff
错误/关键 浅红 #ffc9c9
存储/数据 浅青 #c3fae8

12. humanizer（去除AI写作痕迹）

用途：识别并去除AI生成文本的29种特征模式，使文字听起来自然、有人情味。

五大类AI特征：

内容模式：

  • 无端强调重要性/legacy/趋势（”是一个重要的里程碑”）
  • 无端强调知名度和媒体覆盖
  • 表面分析性的-ing形式（”强调、确保、反映”）
  • 促销/广告式语言（”令人惊叹的”、”鲜艳的”、”位于核心位置”）
  • 模糊归属和措辞（”专家认为”、”行业报告称”）

语言/语法模式：

  • 过度使用的”AI词汇”（actually, crucial, pivotal, showcase, underscore…）
  • 避免使用”是/是”（”作为…的载体”而非”是”）
  • 负面平行结构和尾随否定
  • 三分法滥用（总是把想法分成三组）
  • 优雅变体（同义词循环）

风格模式：

  • 过度使用破折号
  • 过度使用粗体
  • 标题中的标题大小写
  • 表情符号装饰

沟通模式：

  • 协作沟通伪影（”希望这有帮助”、”当然！”）
  • 知识截止免责声明（”截至我的最后训练更新…”）
  • 谄媚/卑屈的语气

填充词和 hedge：

  • 填充短语（”为了实现这一目标”、”由于…的事实”）
  • 过度hedge（”可能潜在地争论该政策可能有一些影响”）
  • 通用正面结论（”未来是光明的”、”令人兴奋的时代就在前面”）

添加灵魂的技巧：

  • 有观点——不要只报道事实，要反应
  • 变化节奏——短促有力的句子，然后是悠长的句子
  • 承认复杂性——真实的人有复杂感受

13. manim-video（3Blue1Brown数学动画）

用途：使用 Manim Community Edition 创建3Blue1Brown风格的数学/算法动画视频。

模式：

  • 概念解释器、方程推导、算法可视化、数据故事、架构图、论文解释器、3D可视化

技术栈：Python 3.10+、Manim CE v0.20+、LaTeX（公式渲染）、ffmpeg（视频合成）

核心原则：

  • 在写代码之前先构思叙事弧线——这个视频要纠正什么误解？”aha moment”是什么？
  • 先展示形状，再展示方程——几何在代数之前
  • 不透明度分层引导注意力：主要元素1.0，背景元素0.4，结构元素0.15

关键实现注意：

  • LaTeX必须用原始字符串：MathTex(r"\frac{1}{2}") 而非 MathTex("\frac{1}{2}")
  • 边缘文本的buff必须>=0.5
  • 替换文本前必须FadeOut
  • 永远不要动画未添加的mobjects

14. p5js（创意编程/生成艺术）

用途：在浏览器中创建交互式和程序生成的视觉艺术，输出为HTML/PNG/GIF/MP4/SVG。

模式：生成艺术、数据可视化、交互体验、动画/运动图形、3D场景、图像处理、音频驱动

关键原则：

  • 禁用FES（Friendly Error System）：p5.disableFriendlyErrors = true — 提升性能
  • 始终使用种子化随机：randomSeed() + noiseSeed() 确保可重复性
  • 使用HSB颜色模式而非RGB
  • 使用 createGraphics() 进行分层合成——单程渲染看起来很平

性能目标：

  • 交互式：60fps
  • 粒子数（P2D形状）：5,000-10,000 at 60fps
  • 粒子数（像素缓冲区）：50,000-100,000 at 60fps

15. pixel-art（像素艺术生成）

用途：将任何图片转换为复古像素艺术，并可选择动画为带时代感特效的短视频。

预设风格：

预设 时代 调色板 块大小 最佳用途
arcade 80年代街机 自适应16色 8px 粗犷海报
snes 16位 自适应32色 4px 角色、精细场景
nes 8位 NES (54色) 8px 真实NES外观
gameboy DMG掌机 4级绿色 8px 单色Game Boy
pico8 PICO-8 16固定 6px 幻想主机外观
c64 Commodore 64 16固定 8px 8位家用电脑

动画场景：night（星星+萤火虫+落叶）、urban（雨+霓虹）、snow（雪花）、fire（余烬）、underwater（气泡）等

16. popular-web-designs（54个设计系统）

用途：当用户想要”做成Stripe的样子”、”做成Linear的样子”时，提供精确的CSS/颜色值。

54个设计模板，分类：

  • AI/机器学习：Claude、Cohere、ElevenLabs、Minimax、Mistral、Ollama、OpenCode、Replicate、RunwayML、Together AI、VoltAgent、xAI
  • 开发者工具：Cursor、Expo、Linear、Lovable、Mintlify、PostHog、Raycast、Resend、Sentry、Supabase、Superhuman、Vercel、Warp、Zapier
  • 基础设施/云：ClickHouse、Composio、HashiCorp、MongoDB、Sanity、Stripe
  • 设计/生产力：Airtable、Cal、Clay、Figma、Framer、Intercom、Miro、Notion、Pinterest、Webflow
  • 金融科技：Coinbase、Kraken、Revolut、Wise
  • 企业/消费：Airbnb、Apple、BMW、IBM、NVIDIA、SpaceX、Spotify、Uber

使用流程：选择设计模板 → skill_view(name="popular-web-designs", file_path="templates/stripe.md") → 获取完整设计令牌 → 在HTML中应用

17. pretext（DOM-free文本布局）

用途：构建创意浏览器演示，使用@chenglou/pretext实现——无DOM的多行文本测量和布局。

核心功能：给定(text, font, width)，返回换行位置、每行宽度、每个字素位置和总高度——全部通过canvas测量，无重排。

创意应用：

  • 文本围绕移动障碍物流动（hero区域、编辑布局）
  • ASCII艺术效果使用真实单词而非单色光栅
  • 文字组成的游戏（字母俄罗斯方块、散文 breakout）
  • 动理学排版，每个字素有物理效果
  • 多行”收缩包裹”UI

两个核心API：

  • prepare(text, font) + layout(prepared, width) → 测量高度（用例1：CSS/DOM渲染）
  • prepareWithSegments(text, font) + layoutWithLines(prepared, width) → 获取每行内容和位置（用例2：自定义canvas渲染）

18. sketch（设计草图变体）

用途：快速生成2-3个设计变体进行比较，用于设计探索和方案对比。

核心原则：

  • 从上下文出发，不凭感觉——先检查现有代码、设计令牌、截图
  • 定义该工件的设计系统：颜色、字体、间距、圆角、阴影、动效姿态、组件处理、交互规则
  • 选择正确的格式：静态视觉比较→一个HTML画布并排；交互/流程→可点击原型；演示→固定大小HTML幻灯片

输出：自包含HTML文件，带变体切换控件。

19. songwriting-and-ai-music（歌曲创作与AI音乐）

用途：歌曲创作技巧 + Suno AI 音乐提示词工程。

核心技能：

  • 歌曲结构：ABABCB、AABA、ABAB、AAA
  • 押韵类型：完美押韵、家族押韵、谐音押韵、辅音押韵、斜押韵
  • 内部押韵：行内押韵，而非只在行尾
  • 情感弧线：能量映射——“从耳语到咆哮再到耳语”

Suno提示词工程：

格式：Genre + Mood + Era + Instruments + Vocal Style + Production + Dynamics

好的例子：
"Cinematic orchestral spy thriller, 1960s Cold War era, smoky sultry female vocalist, 
big band jazz, brass section with trumpets and french horns, sweeping strings, 
minor key, vintage analog warmth"

坏的例子：
"sad rock song"

元标签（放在歌词中）：

  • 结构：[Intro] [Verse] [Pre-Chorus] [Chorus] [Bridge] [Outro]
  • 声乐：[Whispered] [Belted] [Falsetto] [Harmonies]
  • 动态：[Building Energy] [Explosive] [Quiet arrangement]

20. touchdesigner-mcp（TouchDesigner控制）

用途：通过 twozero MCP 服务器控制运行中的 TouchDesigner 实例，创建运算符、设置参数、连接、执行Python、构建实时视觉。

36个原生工具，核心工具：

工具 功能
td_execute_python 在TD中运行任意Python
td_create_operator 创建带参数的节点+自动定位
td_set_operator_pars 安全设置参数（验证，不会崩溃）
td_get_network 查看网络结构
td_get_errors 递归查找错误/警告
td_get_screenshot 捕获一个OP查看器到文件

关键规则：

  • 绝不猜参数名——先调用td_get_par_info
  • 如果tdAttributeError触发，停下来
  • 绝不在脚本回调中硬编码绝对路径——用me.parent() / scriptOp.parent()
  • 优先使用原生MCP工具而非td_execute_python

音频反应GLSL的经验证配方：

  • TimeSlice必须保持ON（否则 AudioSpectrum 处理整个音频文件→24000+样本→溢出）
  • 输出长度手动设置为256
  • 不要在频谱上使用Lag CHOP——它以timeslice模式操作并扩展样本
  • 平滑应在GLSL着色器中进行：mix(prevValue, newValue, 0.3)
